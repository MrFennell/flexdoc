What is Flexbox?

Those who worked on CSS before the age of HTML5 and CSS3 will remember the pains of what should have been simple design layouts. Floats and positioning could only take us so far, and we were often stuck using tricks to make everything fit together on the web page's screen. This was only exacerbated by the rise of smartphone technology and every site being made to fit in multiple screen sizes.

CSS Flex is an exciting, new way to dynamically fit objects inside of a container. The main idea is to let the container alter it's items' width, height and order to best fill the available space within itself. This is huge in the age of smartphone technology where responsive design is a must.

Parent, or Container, properties

When using Flexbox it is important to understand that there are two elements (or groups of elements) in play: the container and the items inside, or the parent and its children. Both can be styled, which is the real power of Flexbox!

Display as Flex

Before the Flexbox layout there were four layout modules we could use:

  • Block
  • Inline

flex-wrap

By default Flex will try and fit everything onto one line. You can change that with the "flex-wrap" property.

flex-wrap : wrap;

This will fit items onto multiple lines depending on the width of the parent property. All options are:

flex-flow

This is simply shorthand for both flex-direction and flex-wrap. Together they define the flex contaner's main and cross axis.

The default setting is:

flex-flow : row nowrap;

This will fit items onto multiple lines depending on the width of the parent property. All options are:

flex-flow: <‘flex-direction’> || <‘flex-wrap’>

justify-content

Set alignment of the main axis with justify-content.

justify-content: flex-start; // (default): items are packed toward the start line justify-content: flex-end; //items are packed toward to end line justify-content: center; //items are centered along the line justify-content: space-between; //items are evenly distributed in the line; first item is on the start line, last item on the end line justify-content: space-around; //items are evenly distributed in the line with equal space around them. Note that visually the spaces aren't equal, since all the items have equal space on both sides. The first item will have one unit of space against the container edge, but two units of space between the next item because that next item has its own spacing that applies. justify-content: space-evenly; //items are distributed so that the spacing between any two items (and the space to the edges) is equal.

align-items

The "align-items" property defines how items are aligned along the cross axis of the current line.

This can be thought of as the "justify-content" version for the cross-axis, which is perpendicular to the main axis.

align-items: flex-start; //cross-start margin edge of the items is placed on the cross-start line align-items: flex-end; //cross-end margin edge of the items is placed on the cross-end line align-items: center; //items are centered along the line align-items: baseline; //items are aligned such as their baselines align align-items: stretch; //(default): stretch to fill the container (still respect min-width/max-width)

align-content

The "align-content" property aligns the items to the container's lines when there is extra space in the cross-axis.

Note that this does nothing when there is only one line of flex items.

align-content: flex-start; //lines packed to the start of the container align-content: flex-end; //lines packed to the end of the container align-content: center; //lines packed to the center of the container align-content: space-between; //lines evenly distributed; the first line is at the start of the container while the last one is at the end align-content: space-around; //lines evenly distributed with equal space around each line align-content: stretch; //(default): lines stretch to take up the remaining space

Child, or Item, Properties

Similarly to how we can style the parent container, the child items within have their own properties that can be styled.

order

The "order" property can change the order that items are laid out in.

By default the order is set to 0.

item{ order: integer; }

flex-grow

Flex items have the capacity to grow and fill in space if desired. Changing the flex-grow property lets this happen.

flex-grow is a unitless value that will change proportionally to all other values. For instance, if all items are set to 1 aside from Item B that is set to 2, Item B will grow twice as large to fill up space.

flex-grow is set to 0 by default and negative numbers are invalid.

item{ flex-grow: ; }

flex-shrink

Similarly to flex-grow, this allows a child element to shrink as necessary in a proportional manner.

flex-shink is set to 1 by default and negative numbers are invalid.

item{ flex-shink: ; }

flex-basis

Similarly to flex-grow, this allows a child element to shrink as necessary in a proportional manner.

flex-shink is set to 1 by default and negative numbers are invalid.

item{ flex-basis: ; //ex: 20%, 4em, 2px flex-basis: auto; //(default), sets to width or height property }

flex (shorthand)

The "flex" property is shorthand for using all of the flex-grow, flex-shrink and flex-basis together.

flex-shrink and flex-basis are optional.

Default is 0 1 auto.

item{ flex: flex-grow flex-shrink flex-basis; flex: 0 1 100%; //ex }

align-self

The "align-self" property allows items to overwrite the default or parent alignment to be overridden.

Uses the same values as the align-self property.

Default is 0 1 auto.

item{ flex: flex-grow flex-shrink flex-basis; flex: 0 1 100%; //ex }
